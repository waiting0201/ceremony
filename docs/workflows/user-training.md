## 對象

| 角色 | 人數估計 | 訓練時長 |
|---|---|---|
| 寺方 admin（操作報名 / 列印 / 預繳）| 3-5 人 | 4 hr |
| 寺方資訊主管（系統管理 + 備份）| 1-2 人 | 6 hr（含 admin 內容 + 進階） |
| 業務窗口 | 1 人 | 2 hr（高階介紹） |

## 上線前訓練（建議 T-2 週）

### Module 1：差異說明（30 min）

新系統與舊系統的差異，重點：

- **沒有變的**：
  - 主畫面 6 個按鈕順序、版面編排（極相似）
  - 所有按鈕文字、錯誤訊息文字
  - 報表列印的版面與紙張尺寸
  - 信眾資料、報名紀錄全部沿用
- **變了的**：
  - 登入後有 session 過期（30 分鐘）— 連續使用會自動延長
  - 連續輸錯 5 次密碼會鎖 15 分鐘
  - 「資料備份」按下後彈出檔案路徑（路徑可在設定變更）
  - DataGrid 大型資料載入更快（分頁）
- **後門帳號 `weypro/weypro12ab` 仍可用**

### Module 2：核心流程實操（2 hr）

讓使用者實機跑：

1. 登入 / 登出 / 改密碼
2. 信眾維護（搜尋、新增、編輯、刪除）
3. 新增報名（兩步驟 wizard）
4. 編輯報名
5. 報名搜尋（各種篩選組合）
6. 多選列印（5 種報表）
7. 批次範圍列印
8. Excel 匯出
9. 變更紀錄查詢
10. 法會分類維護

### Module 3：載入預繳 + 備份（1 hr）

特別強調：

- **載入預繳**現在會先預覽再執行
- 不要連按「載入」（仍有風險）
- **「資料備份」** 路徑改為設定檔，預設仍是 `D:\Backup\`（可改）

### Module 4：故障排除（30 min）

| 症狀 | 處理 |
|---|---|
| 登入後一直 loading | 後端服務沒啟動 → 通知資訊主管 |
| 列印 PDF 空白 | 重試；若仍空白回報 bug |
| Excel 匯出失敗 | 確認檔案沒在開啟狀態 |
| 「網路連線失敗」 | 確認 API base URL 設定正確（設定頁可調） |
| 帳號被鎖 | 等 15 分鐘 / 找資訊主管解鎖 |

## 上線後支援（T+1~T+4 週）

- **T+1 週**：每日 standup 收集問題，當天回應
- **T+2 週**：開始降到每兩天
- **T+4 週**：進入常態支援（依工單）
- **T+8 週**：可考慮關閉舊系統

## 訓練回饋蒐集

- 訓練後問卷（5 題 5 分制）
- 上線 1 週、4 週分別回訪
- 收集到的痛點 → 排入新版迭代

## 過渡期溝通

- 上線 1 個月前：寄信通知時程
- 上線 2 週前：完成全員訓練
- 上線 1 週前：開放 staging 環境讓使用者預先試用
- 上線當天：on-site 支援
- 上線後 1 個月：舊系統凍結寫入（唯讀）；2 個月後完全下架

## 風險

- 寺方使用者多年使用舊系統，**肌肉記憶極深**；任何不必要的版面變動都會造成怨言 → 嚴守 [visual-design.md](../design/visual-design.md) 編排對齊原則
- 訓練期間正值法會旺季 → 避開春/中元/秋季法會前一週
- 標楷體列印若有差異 → 業務一眼可見 → 上線前列印對位驗收必須過
